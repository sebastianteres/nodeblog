About Me

Projects

My full name is Sebastian Perez Teres, I’m a creative frontend engineer focused on delivering usable and beautiful applications to the end user.
I have 4 years of experience working on amazing applications with a challenging user experience.
I like well organized, maintainable and testable code. I have been working in Agile environments all this time using tools like trello and Asana to keep top productivity.

Most of all I like creating reusable code and experiences, take a custom feature and, when possible and if it makes sense, make it configurable enough to be used in different situations by different users.

I have been working remotely from Mexico for 4 years, mostly for Startups.
Working from home wasn’t easy at first but over the time I’ve learned to organize my time and tasks so to be as productive, or more, as working on site.

I’m always looking for an exciting project to work on, usually just to learn about a technology or language. I try to keep myself up to date with the latest news, technologies and libraries out there, if I see something interesting, I’ll give it a try and see if it would help on current projects.

My first baby

This awesome startup consisted on building a web CMS running on web browsers but targeting mobile devices.
The main objective was to make it as simple as possible, designed by the greatest UX Engineer I’ve ever met, we built an
Edit-In-Place, Drag and Drop editor (I was in charge of frontend). Users absolutely loved it!

Later on project evolved to adapt clients SaaS and somehow automagically create mobile web apps based on their API.
This time I was mainly in charge of the backend writing web services in Grails, using a content repository (JCR) to store most of the data.

Optimize, Optimize!

Based on NYC, they offer project management and time tracking solutions for any kind of business.

This huge web app built in .NET has a critical page, the time entry tool. That page was particullary slow so I jumped in to help make it faster. Brilliant is often challenging

The idea is simple, help publishers charge for their content. The solution, however, hides all kinds of backbreaking efforts that comes with launching a widget in a thirdparty site.

Of course everything needs to be super fast and secure. They also offer an elegant console for publishers to set up their account and start making some money.
I was in charge of building an Edit-In-Place Editor to customize the subscription widget.

I was also able to help with mediapass' iOS SDK they offer to publishers that wishes to use mediapass on their iOS app. First by fixing a few bugs and later on extending it.
